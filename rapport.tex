\chapter{Introduction}

\section{Contexte}
La cryptographie est la science de la protection de l'information par le biais de techniques mathématiques. Elle joue un rôle fondamental dans la sécurisation des communications numériques modernes, que ce soit pour le chiffrement des données, l'authentification des utilisateurs ou la vérification de l'intégrité des messages.

\section{Objectifs du projet}
Ce projet a pour objectif l'implémentation en Python des principaux algorithmes cryptographiques étudiés en cours, couvrant :
\begin{itemize}
    \item Les \textbf{chiffrements classiques} : César, Affine, Vigenère, Hill, Playfair, OTP, Substitution aléatoire
    \item Les \textbf{outils de cryptanalyse} : Analyse fréquentielle, Indice de coïncidence, Test de Kasiski
    \item Le \textbf{chiffrement symétrique (synchrone)} : DES, 2DES, 3DES, DESX, AES (ECB/CBC/CTR), RC4
    \item Le \textbf{chiffrement asymétrique} : RSA, ElGamal, Diffie-Hellman, ECC
    \item Les \textbf{fonctions de hachage} : MD5, SHA-256, SHA-512, HMAC
    \item Les \textbf{signatures numériques} : RSA, ElGamal, DSA, ECDSA
    \item Un \textbf{canal de communication sécurisé} : Client/Serveur avec RSA + AES-GCM
\end{itemize}

\section{Structure du projet}
Le projet est organisé en répertoires thématiques :

% --- CESAR ---
\section{Chiffrement de César}

\subsection{Principe}
Le chiffrement de César est un chiffrement par substitution mono-alphabétique. Chaque lettre du texte clair est décalée d'un nombre fixe de positions dans l'alphabet.

\textbf{Formules :}
\begin{align}
\text{Chiffrement :} \quad E(x) &= (x + k) \mod 26 \\
\text{Déchiffrement :} \quad D(x) &= (x - k) \mod 26
\end{align}

\subsection{Code source}
\begin{lstlisting}[style=python,caption=Cesar.py - Fonctions principales]
def cesar_encrypt(text: str, shift: int) -> str:
    result = ""
    for char in text:
        if char.isalpha():
            ascii_offset = ord('A') if char.isupper() else ord('a')
            result += chr((ord(char) - ascii_offset + shift) % 26
                         + ascii_offset)
        else:
            result += char
    return result

def cesar_decrypt(text: str, shift: int) -> str:
    return cesar_encrypt(text, -shift)
\end{lstlisting}

\subsection{Fonctionnement}
Le programme propose un menu interactif permettant de chiffrer ou déchiffrer un texte avec une clé entière positive. Le déchiffrement réutilise la fonction de chiffrement avec un décalage négatif.

% --- AFFINE ---
\section{Chiffrement Affine}

\subsection{Principe}
Le chiffrement affine utilise une fonction affine pour le chiffrement :
\begin{align}
E(x) &= (a \cdot x + b) \mod 26 \\
D(y) &= a^{-1} \cdot (y - b) \mod 26
\end{align}
où $a$ doit être premier avec 26 (i.e. $\gcd(a, 26) = 1$).

\subsection{Code source}
\begin{lstlisting}[style=python,caption=affine.py - Fonctions principales]
def affine_encrypt(text: str, a: int, b: int) -> str:
    result = ""
    for char in text:
        if char.isalpha():
            ascii_offset = ord('A') if char.isupper() else ord('a')
            x = ord(char) - ascii_offset
            y = (a * x + b) % 26
            result += chr(y + ascii_offset)
        else:
            result += char
    return result

def affine_decrypt(text: str, a: int, b: int) -> str:
    try:
        a_inv = pow(a, -1, 26)
    except ValueError:
        raise ValueError("L'inverse n'existe pas")
    result = ""
    for char in text:
        if char.isalpha():
            ascii_offset = ord('A') if char.isupper() else ord('a')
            y = ord(char) - ascii_offset
            x = (a_inv * (y - b)) % 26
            result += chr(x + ascii_offset)
        else:
            result += char
    return result
\end{lstlisting}

L'inverse modulaire est calculé avec l'algorithme d'Euclide étendu via \texttt{pow(a, -1, 26)}.

% --- HILL ---
\section{Chiffrement de Hill}

\subsection{Principe}
Le chiffrement de Hill utilise l'algèbre linéaire. Le texte est découpé en vecteurs de taille $n$ et multiplié par une matrice-clé $K$ de taille $n \times n$ modulo 26. Le déchiffrement utilise la matrice inverse $K^{-1} \mod 26$.

\subsection{Code source}
\begin{lstlisting}[style=python,caption=Hill.py - Chiffrement]
def hill_encrypt(text: str, key_matrix: np.ndarray) -> str:
    n = key_matrix.shape[0]
    text = ''.join(c for c in text if c.isalpha()).lower()
    if len(text) % n != 0:
        text += 'x' * (n - len(text) % n)  # padding
    vectors = []
    for i in range(0, len(text), n):
        vector = [ord(text[i + j]) - ord('a') for j in range(n)]
        vectors.append(vector)
    result = ""
    for vector in vectors:
        encrypted_vector = np.dot(key_matrix, vector) % 26
        for num in encrypted_vector:
            result += chr(int(num) + ord('a'))
    return result

def hill_decrypt(text: str, key_matrix: np.ndarray) -> str:
    inverse_key = matrix_mod_inverse(key_matrix, 26)
    return hill_encrypt(text, inverse_key)
\end{lstlisting}

% --- PLAYFAIR ---
\section{Chiffrement de Playfair}

\subsection{Principe}
Le chiffrement de Playfair utilise une matrice 5$\times$5 construite à partir d'un mot-clé. Le texte est découpé en bigrammes, et chaque paire est chiffrée selon trois règles : même ligne, même colonne, ou rectangle.

\subsection{Code source}
\begin{lstlisting}[style=python,caption=Playfair.py - Construction de la matrice]
def create_playfair_matrix(key: str) -> List[List[str]]:
    key = key.upper().replace('J', 'I')
    alphabet = []
    for char in key + 'ABCDEFGHIKLMNOPQRSTUVWXYZ':
        if char not in alphabet and char.isalpha():
            alphabet.append(char)
    matrix = []
    for i in range(0, 25, 5):
        matrix.append(alphabet[i:i + 5])
    return matrix
\end{lstlisting}

% --- OTP ---
\section{One-Time Pad (OTP)}

\subsection{Principe}
Le One-Time Pad est le seul chiffrement théoriquement incassable. Il utilise une clé aléatoire de même longueur que le message et effectue un XOR bit à bit.

\subsection{Code source}
\begin{lstlisting}[style=python,caption=OTP.py]
import os

def otp_encrypt(message: str) -> tuple[bytes, bytes]:
    message_bytes = message.encode("utf-8")
    key = os.urandom(len(message_bytes))
    ciphertext = bytes(m ^ k for m, k in zip(message_bytes, key))
    return ciphertext, key

def otp_decrypt(ciphertext: bytes, key: bytes) -> str:
    message_bytes = bytes(c ^ k for c, k in zip(ciphertext, key))
    return message_bytes.decode("utf-8")
\end{lstlisting}

import random, string
from typing import Dict

def generate_random_substitution() -> Dict[str, str]:
    alphabet = string.ascii_lowercase
    shuffled = list(alphabet)
    random.shuffle(shuffled)
    return {alphabet[i]: shuffled[i] for i in range(len(alphabet))}

def substitution_encrypt(text, substitution_table):
    result = ""
    for char in text.lower():
        if char in substitution_table:
            result += substitution_table[char]
        else:
            result += char
    return result

\chapter{Outils de Cryptanalyse}

\section{Analyse Fréquentielle}

\subsection{Principe}
L'analyse fréquentielle compare la fréquence des lettres d'un texte chiffré aux fréquences théoriques de la langue (ici le français). Cela permet de casser les chiffrements par substitution mono-alphabétique.

\subsection{Code source}
\begin{lstlisting}[style=python,caption=AnalyseFrequentielle.py]
FRENCH_FREQ = {
    'A': 7.636, 'B': 0.901, 'C': 3.260, 'E': 14.715, ...
}

def analyse_frequentielle(texte):
    texte = normalize_text(texte)
    lettres = [c for c in texte if c in string.ascii_lowercase]
    total = len(lettres)
    compte = Counter(lettres)
    frequences = {l: (compte[l] / total) * 100 for l in compte}
    return frequences
\end{lstlisting}
Le programme génère aussi un histogramme comparatif avec \texttt{matplotlib}.

\section{Indice de Coïncidence}

\subsection{Principe}
L'indice de coïncidence (IC) mesure la probabilité que deux lettres choisies au hasard soient identiques :
$$IC = \frac{\sum_{i=1}^{26} n_i(n_i - 1)}{N(N-1)}$$

\subsection{Code source}
\begin{lstlisting}[style=python,caption=IndiceDeCoincidence.py]
def indice_coincidence(text):
    text = normalize_text(text)
    letters = [c for c in text if c in string.ascii_lowercase]
    N = len(letters)
    if N <= 1:
        return 0
    counts = Counter(letters)
    somme = sum(n * (n - 1) for n in counts.values())
    return somme / (N * (N - 1))
\end{lstlisting}

\section{Test de Kasiski}

\subsection{Principe}
Le test de Kasiski permet d'estimer la longueur de la clé d'un chiffrement de Vigenère en recherchant les séquences répétées et en calculant le PGCD des distances entre ces répétitions.

\subsection{Code source}
\begin{lstlisting}[style=python,caption=TestDeKasiski.py - Fonctions clés]
def find_repeated_sequences(ciphertext, min_length=3):
    sequences = {}
    for i in range(len(ciphertext) - min_length + 1):
        seq = ciphertext[i:i + min_length]
        if seq in sequences:
            sequences[seq].append(i)
        else:
            sequences[seq] = [i]
    gaps = []
    for seq, positions in sequences.items():
        if len(positions) > 1:
            for j in range(1, len(positions)):
                gaps.append(positions[j] - positions[j - 1])
    return gaps

def find_key_length(ciphertext):
    gaps = find_repeated_sequences(ciphertext)
    if not gaps:
        return None
    return reduce(gcd, gaps)
\end{lstlisting}

\chapter{Chiffrement Symétrique (Synchrone)}

\section{DES (Data Encryption Standard)}

\subsection{Principe}
Le DES est un algorithme de chiffrement par blocs de 64 bits avec une clé de 56 bits. Il utilise un réseau de Feistel à 16 rounds. Notre implémentation inclut également le 2DES, 3DES et DESX.

\subsection{Architecture}
L'algorithme se décompose en :
\begin{enumerate}
    \item \textbf{Permutation initiale (IP)} : réarrangement des 64 bits
    \item \textbf{16 rounds de Feistel} : chaque round applique expansion, XOR avec sous-clé, S-Boxes et permutation P
    \item \textbf{Permutation finale (FP)} : inverse de IP
\end{enumerate}

\subsection{Code source}
\begin{lstlisting}[style=python,caption=DES.py - Fonction de Feistel]
def feistel(right, subkey):
    expanded = permute(right, E)  # 32 -> 48 bits
    xored = xor_bits(expanded, subkey)
    sbox_output = []
    for i in range(8):
        block = xored[i*6:(i+1)*6]
        row = (block[0] << 1) | block[5]
        col = (block[1]<<3)|(block[2]<<2)|(block[3]<<1)|block[4]
        sbox_output.extend(
            [int(b) for b in format(S_BOXES[i][row][col], '04b')]
        )
    return permute(sbox_output, P)
\end{lstlisting}

\begin{lstlisting}[style=python,caption=DES.py - Chiffrement d'un bloc]
def des_encrypt_block(block_64bits, subkeys):
    state = permute(block_64bits, IP)
    L, R = state[:32], state[32:]
    for i in range(16):
        new_L = R
        new_R = xor_bits(L, feistel(R, subkeys[i]))
        L, R = new_L, new_R
    return permute(R + L, FP)
\end{lstlisting}

\begin{lstlisting}[style=python,caption=DES.py - Triple DES]
def _3des_encrypt(plaintext, key1, key2, key3=None):
    """3DES: E_k1(D_k2(E_k3(plaintext)))"""
    if key3 is None:
        key3 = key1  # Mode 2-cles: K1=K3
    encrypted = des_encrypt(plaintext, key3)
    decrypted = des_decrypt(encrypted, key2)
    return des_encrypt(decrypted, key1)
\end{lstlisting}

\subsection{Variantes implémentées}
\begin{itemize}
    \item \textbf{DES simple} : chiffrement ECB avec blocs de 64 bits
    \item \textbf{2DES} : double application du DES avec deux clés
    \item \textbf{3DES (EDE)} : triple application Encrypt-Decrypt-Encrypt
    \item \textbf{DESX} : ajout de whitening keys (XOR avant/après DES)
\end{itemize}

\section{AES (Advanced Encryption Standard)}

\subsection{Principe}
L'AES est le standard actuel de chiffrement symétrique. Il opère sur des blocs de 128 bits avec des clés de 128, 192 ou 256 bits. Les opérations principales sont : SubBytes, ShiftRows, MixColumns et AddRoundKey.

\subsection{Code source}
Notre implémentation a évolué pour utiliser la bibliothèque professionnelle \texttt{pycryptodome}, offrant des performances optimales et le support des modes ECB, CBC et CTR, tout en corrigeant les vulnérabilités de notre première approche manuelle.

\begin{lstlisting}[style=python,caption=AES.py - Chiffrement CBC]
from Crypto.Cipher import AES
from Crypto.Util.Padding import pad, unpad
from Crypto.Random import get_random_bytes

def aes_encrypt_cbc(data, key, iv=None):
    """AES mode CBC."""
    if isinstance(data, str):
        data = data.encode('utf-8')
    if isinstance(key, str):
        key = key.encode('utf-8')[:32].ljust(32, b'\x00')
    if iv is None:
        iv = get_random_bytes(AES.block_size)

    cipher = AES.new(key, AES.MODE_CBC, iv=iv)
    padded = pad(data, AES.block_size)
    ciphertext = cipher.encrypt(padded)
    return iv + ciphertext
\end{lstlisting}

\subsection{Modes implémentés}
\begin{itemize}
    \item \textbf{ECB} : chaque bloc chiffré indépendamment
    \item \textbf{CBC} : chaînage avec XOR du bloc précédent
    \item \textbf{CTR} : mode compteur (chiffrement de flux)
\end{itemize}

\section{RC4 (Rivest Cipher 4)}

\subsection{Principe}
RC4 est un chiffrement de flux (stream cipher). Il génère un flux pseudo-aléatoire (keystream) via deux algorithmes : KSA (Key Scheduling Algorithm) et PRGA (Pseudo-Random Generation Algorithm).

\subsection{Code source}
\begin{lstlisting}[style=python,caption=RC4.py - Algorithmes KSA et PRGA]
class RC4:
    def __init__(self, key):
        if isinstance(key, str):
            key = key.encode('utf-8')
        self.key = key
        self.S = list(range(256))
        self._ksa(key)

    def _ksa(self, key):
        """Key Scheduling Algorithm"""
        key_length = len(key)
        j = 0
        for i in range(256):
            j = (j + self.S[i] + key[i % key_length]) % 256
            self.S[i], self.S[j] = self.S[j], self.S[i]

    def _prga(self):
        """Pseudo-Random Generation Algorithm"""
        self._i = (self._i + 1) % 256
        self._j = (self._j + self.S[self._i]) % 256
        self.S[self._i], self.S[self._j] = self.S[self._j], self.S[self._i]
        return self.S[(self.S[self._i] + self.S[self._j]) % 256]

    def encrypt(self, data):
        if isinstance(data, str):
            data = data.encode('utf-8')
        self._i = 0; self._j = 0
        self.S = list(range(256))
        self._ksa(self.key)
        result = bytearray()
        for byte in data:
            result.append(byte ^ self._prga())
        return bytes(result)
\end{lstlisting}

\subsection{Variantes implémentées}
\begin{itemize}
    \item \textbf{RC4 Standard} : implémentation classique
    \item \textbf{RC4A} : variante avec deux S-boxes
    \item \textbf{RC4Drop} : saute les N premiers bytes du keystream
    \item \textbf{RC4WithHash} : clé dérivée via SHA-256
\end{itemize}

\chapter{Chiffrement Asymétrique}

\section{RSA}

\subsection{Principe}
RSA repose sur la difficulté de factoriser de grands nombres. On choisit $p, q$ premiers, $n = p \cdot q$, $\phi = (p-1)(q-1)$, $e$ tel que $\gcd(e, \phi) = 1$ et $d = e^{-1} \mod \phi$.
\begin{align}
\text{Chiffrement :} \quad c &= m^e \mod n \\
\text{Déchiffrement :} \quad m &= c^d \mod n
\end{align}

\subsection{Code source}
Le code a évolué d'une implémentation pédagogique basique vers l'utilisation de \texttt{cryptography} pour intégrer le padding sécurisé OAEP, essentiel contre les attaques sur RSA textbook, ainsi qu'une approche de chiffrement hybride pour de larges volumes de données.

\begin{lstlisting}[style=python,caption=RSA.py - Génération et Chiffrement OAEP]
from cryptography.hazmat.primitives.asymmetric import rsa, padding
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.backends import default_backend

def generate_rsa_key_pair(key_size: int = 2048):
    private_key = rsa.generate_private_key(
        public_exponent=65537,
        key_size=key_size,
        backend=default_backend()
    )
    return private_key, private_key.public_key()

def rsa_encrypt_oaep(message: bytes, public_key) -> bytes:
    return public_key.encrypt(
        message,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
\end{lstlisting}

\section{ElGamal}

\subsection{Principe}
ElGamal est basé sur le problème du logarithme discret. On choisit un nombre premier $p$, un générateur $g$, une clé privée $a$ et la clé publique $A = g^a \mod p$.
\begin{align}
\text{Chiffrement :} \quad c_1 &= g^k \mod p, \quad c_2 = m \cdot A^k \mod p \\
\text{Déchiffrement :} \quad m &= c_2 \cdot (c_1^a)^{-1} \mod p
\end{align}

\subsection{Code source}
\begin{lstlisting}[style=python,caption=ElGamal.py - Chiffrement/Dechiffrement]
def encrypt(message, public_key):
    p, g, A = public_key
    cipher = []
    for char in message:
        m = ord(char)
        k = random.randint(2, p - 2)
        c1 = pow(g, k, p)
        c2 = (m * pow(A, k, p)) % p
        cipher.append((c1, c2))
    return cipher

def decrypt(cipher, private_key, p):
    a = private_key
    message = ""
    for c1, c2 in cipher:
        s = pow(c1, a, p)
        s_inv = mod_inverse(s, p)  # Fermat: a^(p-2) mod p
        m = (c2 * s_inv) % p
        message += chr(m)
    return message
\end{lstlisting}

\section{Diffie-Hellman}

\subsection{Principe}
Diffie-Hellman permet à deux parties d'établir un secret partagé via un canal non sécurisé. Avec $p$ premier et $g$ racine primitive :
\begin{enumerate}
    \item Alice choisit $a$, calcule $A = g^a \mod p$
    \item Bob choisit $b$, calcule $B = g^b \mod p$
    \item Secret partagé : $s = B^a \mod p = A^b \mod p$
\end{enumerate}

\subsection{Principe}
La cryptographie sur les courbes elliptiques (ECC) offre un niveau de sécurité équivalent à RSA mais avec des clés beaucoup plus petites. Elle repose sur la difficulté du logarithme discret sur le groupe des points d'une courbe elliptique sur un corps fini.

\section{Principe général}
Les fonctions de hachage produisent une empreinte de taille fixe à partir d'un message de taille quelconque. Propriétés essentielles : résistance aux collisions, résistance à la pré-image, effet avalanche.

\section{MD5}

def md5_hash(data):

\section{Principe}
Une signature numérique garantit l'authenticité et la non-répudiation d'un message. L'expéditeur signe avec sa clé privée, le destinataire vérifie avec la clé publique.

class RSASignature:
    def __init__(self, key_size=2048):
        self.private_key = rsa.generate_private_key(
            public_exponent=65537, key_size=key_size)
        self.public_key = self.private_key.public_key()

    def sign_pkcs1v15(self, message):
        return self.private_key.sign(
            message, padding.PKCS1v15(), hashes.SHA256())

    def verify_pkcs1v15(self, message, signature):
        try:
            self.public_key.verify(
                signature, message,
                padding.PKCS1v15(), hashes.SHA256())
            return True
        except Exception:
            return False

\chapter{Questions et Réponses}

\section{Limites pratiques de l'OTP (One-Time Pad)}
\textbf{Question :} L'OTP est théoriquement parfait mais pratiquement inutilisable, citer des obstacles concrets.\\
\textbf{Réponse :}
Bien que l'OTP offre un secret parfait de Shannon, son utilisation en pratique se heurte à plusieurs obstacles majeurs :
\begin{itemize}
    \item \textbf{Taille de la clé :} La clé doit être au moins aussi longue que le message à chiffrer ($|K| = |M|$).
    \item \textbf{Génération de l'aléa :} La clé doit être générée de manière parfaitement aléatoire (entropie maximale) et non pseudo-aléatoire.
    \item \textbf{Distribution des clés :} Échanger de manière sécurisée une clé de la taille du message est logistiquement complexe, car cela nécessite un canal de communication tout aussi sécurisé.
    \item \textbf{Usage unique :} La clé ne doit être utilisée qu'une seule fois. Toute réutilisation compromet immédiatement le système (two-time pad).
\end{itemize}

\section{Impact de la longueur de la clé et lien avec l'OTP}
\textbf{Question :} En quoi la longueur de la clé impacte-t-elle la sécurité ? Que se passe-t-il quand $|K| = |M|$ ? (lien avec OTP)\\
\textbf{Réponse :}
La longueur de la clé détermine la taille de l'espace des clés. Plus la clé est longue, plus il y a de combinaisons possibles, rendant les attaques par force exhaustive irréalisables avec la puissance de calcul actuelle ou future.

Lorsque la longueur de la clé est égale à la longueur du message ($|K| = |M|$), comme dans le cas de l'OTP (avec une clé aléatoire et à usage unique), on atteint la \textbf{sécurité parfaite}. Dans cette configuration, chaque chiffré peut correspondre à n'importe quel texte clair possible de la même longueur. L'attaquant, même avec une puissance de calcul infinie, ne peut déduire aucune information sur le message d'origine à partir du chiffré.

\section{Vulnérabilité du chiffrement de Hill}
\textbf{Question :} Pourquoi Hill est-il vulnérable à l'attaque à clair connu même pour de grandes matrices ?\\
\textbf{Réponse :}
Le chiffrement de Hill est basé sur l'algèbre linéaire ($C = K \cdot P \pmod{26}$).
Dans une attaque à clair connu, l'attaquant n'a pas besoin de tester toutes les matrices possibles par force brute. S'il intercepte suffisamment de paires pour former une matrice $P$ inversible de taille $n \times n$ (où $n$ est la taille de la matrice clé $K$), il peut simplement résoudre l'équation :
$$K = C \cdot P^{-1} \pmod{26}$$
La résolution de ce système d'équations (inversion de matrice) se fait en temps polynomial. Ainsi, même pour de très grandes matrices, retrouver la clé $K$ est très rapide.

\section{Chiffrement RSA et Padding OAEP}
\textbf{Question :} Pourquoi RSA ne peut pas chiffrer directement un message de taille arbitraire ? Qu'est-ce que le padding OAEP apporte par rapport à RSA textbook ?\\
\textbf{Réponse :}
\textbf{Taille du message :} L'algorithme RSA opère dans l'anneau des entiers modulo $n$. Par définition mathématique, la valeur numérique du message $m$ doit être strictement inférieure à $n$. Chiffrer un message plus grand nécessiterait de le découper en blocs, ce qui est inefficace et vulnérable structurellement (similaire au mode ECB). C'est pourquoi RSA est généralement utilisé pour chiffrer des petites données (comme des clés AES symétriques).

\textbf{Apport de l'OAEP :}
Le "RSA textbook" (pur) est déterministe (un même message donne toujours le même chiffré) et malléable.
L'OAEP (Optimal Asymmetric Encryption Padding) apporte deux propriétés fondamentales :
\begin{itemize}
    \item \textbf{Randomisation :} Il introduit un aléa cryptographique (salt) avant le chiffrement, rendant RSA probabiliste (sémantiquement sûr).
    \item \textbf{Contre la malléabilité :} Il structure mathématiquement le message avec des fonctions de hachage, empêchant les attaques par chiffré choisi. Toute modification du chiffré sera détectée lors du déchiffrement.
\end{itemize}

\section{Sélection de Rijndael pour l'AES face à Serpent}
\textbf{Question :} L’algorithme Serpent a obtenu la meilleure note pour la sécurité mais n'a pas été retenu. Quel critère a fait pencher la balance vers Rijndael ?\\
\textbf{Réponse :}
Lors du concours AES organisé par le NIST, Serpent offrait effectivement la marge de sécurité la plus élevée (notamment grâce à ses 32 rounds). Cependant, Rijndael a été sélectionné grâce à son \textbf{excellent équilibre global}, et plus particulièrement pour ses \textbf{performances} et son \textbf{agilité} :
\begin{itemize}
    \item \textbf{Performances :} Rijndael était significativement plus rapide en implémentation logicielle et matérielle. La complexité de Serpent impactait lourdement les débits de chiffrement.
    \item \textbf{Agilité :} Rijndael fonctionnait extrêmement bien sur un large panel de plateformes, des processeurs performants aux puces intégrées (smartcards) disposant de très peu de mémoire (RAM) et de capacités limitées.
\end{itemize}
